### Для products: 

* Частые операции чтения, редкие для записи. Read Repair - подходящий выбор.
* Так нечасто нужно совершать Anti-Entropy Repair (раз в сутки или несколько раз в неделю), для исключения расхождений

### Для orders:
*  Критичны write-операции → Hinted Handoff + Read Repair + регулярный анти-энтропийный репэйр.

### Для carts: 
* частые обновления → Hinted Handoff + фоновый анти-энтропийный репэйр; Read Repair выключен из-за возможного увеличения latency.
